EmojiBoard

Code: github.com/bt1k/emojiboard

React TypeScript Go

EmojiBoard is a very simple website which allows you to submit emojis for others to see. Submitted emojis are stored in a PostgreSQL database. Features include:

  • Rate limiting to reduce server load in the event of many requests.
  • Database check constraint to ensure only certain emojis can be submitted.
  • Database trigger function to ensure that a maximum of 10 emoji posts are in the database at any time (to prevent unlimited growth of database size).

Other projects

The websites below can each take around 1 minute to load. I am using free hosting on Render which puts websites to sleep when inactive.

New Rails Blog

Ruby Rails

New Rails Blog is a simple blog website. Features include:

  • The Trix WYSIWYG text editor for writing rich text blog posts.
  • User authentication, which is required to submit a blog post.
  • A serverless PostgreSQL database hosted by Neon, while images (submitted within blog posts) are hosted by Cloudinary.

The reason for "new" in the name is because I made this website to replace a previous inferior blog website which I also made with Rails.

PostBoard

JavaScript Node.js Preact

PostBoard is similar to EmojiBoard above. Instead of submitting only emojis, you can submit text messages, which are automatically moderated by Google's Gemini AI assistant. Features include:

  • Google's Gemini API is asked, with a suitable prompt, whether each submitted text message is offensive. If a message is offensive, it is not saved into the database.
  • Unlike EmojiBoard above which has a back-end written in Go, PostBoard has a back-end written using Node.js, the web framework Express, and the Prisma ORM.
  • Instead of using React on the front-end, Preact is used. Preact is a lightweight alternative to React, but the developer experience is nearly identical.

Messenger

React Native TypeScript Go

Messenger is a simple tech demo using WebSockets and React Native. Features include:

  • The front-end uses a WebSocket for sustained bidirectional communication between the user's device and the server.
  • React Native is used so that the same codebase can be used to produce a website, a native iOS app, and a native Android app.
  • Unlike the other projects on this page, Messenger does not use a database. Messages are not persisted at all; the server simply sends a received message back to the client.
